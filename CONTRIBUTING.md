# Contributing to SimGrid

This file is a map to our documentation, so that you can quickly find where to look before making changes. We keep it short on
purpose: everything below is explained in more detail, and kept up to date, in the linked files rather than duplicated here.

## Start here, depending on what you need

- **Understand what SimGrid is for**, as a user: [docs/source/Introduction.rst](docs/source/Introduction.rst).
- **Find an example to use as a starting point for your code**, as a user: [examples/README.rst](examples/README.rst)
- **Understand how SimGrid works internally** (actors, activities, resources, the maestro, simcalls, the solver):
  [docs/source/Design_goals.rst](docs/source/Design_goals.rst).
- **Find your way in the source tree** (source tree layout, i.e. which directory implements what): [src/README.rst](src/README.rst)
- **Set up your build for development, coding standards, tests, CI**: the rest of
  [docs/source/Contributors_Documentation.rst](docs/source/Contributors_Documentation.rst).
- **Build/install SimGrid**: [docs/source/Installing_SimGrid.rst](docs/source/Installing_SimGrid.rst).
- **Full documentation index**: [docs/source/index.rst](docs/source/index.rst), also published at <https://simgrid.org/doc/latest/>.

## Before you send a patch

Read the "Enforcing the coding standards", "Interacting with the tests" and "Interacting with git" sections of
[docs/source/Contributors_Documentation.rst](docs/source/Contributors_Documentation.rst). It covers formatting, how/where to add
tests, and our git conventions.

## Where to ask

See the "More information" section of [README.md](README.md) for the Mattermost channel.
